# Rubrics (starter)

## Release notes rubric (example)

Required:
- Summary
- Customer impact
- Breaking changes (explicitly say “None” if none)
- Links to source items (tickets/PRs)

Forbidden:
- Unqualified certainty when the source is missing
- New features not present in input

Scoring (example):
- 0–2: unsafe / invented / misleading
- 3–4: mostly correct, needs edits
- 5: publishable with minimal edits
